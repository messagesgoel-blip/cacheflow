### Remote URL Import (RIMPORT-1)

**Files changed:**

| File | Change |
|------|--------|
| `lib/transfers/remoteUpload.ts` | Added SSRF protection (`validateRemoteUrl`), file size limit (10 GB), extended timeout to 5 min, pre- and post-download size checks |
| `api/src/routes/remoteUpload.js` | Added auth middleware, provider allowlist validation, delegates URL validation to service layer |
| `api/src/services/remoteUploadService.js` | Added `validateRemoteUrl` with private IP blocking, size limits, exported for route-level use |

**Security measures:**
- Private/internal IP blocking (localhost, 10.x, 172.16-31.x, 192.168.x, link-local, IPv6 loopback/ULA)
- DNS resolution validation (`validateResolvedIPs`) ‚Äî resolves hostname and checks all A/AAAA records against private IP ranges before connecting
- Manual redirect handling (`redirect: 'manual'`) ‚Äî validates each redirect hop's URL and resolved IPs to prevent SSRF via open redirects
- Protocol restricted to HTTP/HTTPS only
- File size enforced both from Content-Length header and actual download size
- Auth middleware required on all endpoints
- Provider allowlist: google, onedrive, dropbox, box, pcloud, filen, yandex, vps, webdav, local
- Both TS (`lib/transfers/remoteUpload.ts`) and JS (`api/src/services/remoteUploadService.js`) layers share aligned SSRF protections
