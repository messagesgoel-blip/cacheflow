# Contract: 0.2
## Endpoint or File Path
- `lib/errors/ErrorCode.ts`
- `lib/errors/AppError.ts`

## Request / Response Shape
- Constructor input (`AppErrorOptions`):
  - `code: ErrorCode`
  - `message?: string`
  - `statusCode?: number`
  - `retryable?: boolean`
  - `details?: Record<string, unknown>`
  - `cause?: unknown`
- Serialized output (`SerializedAppError`, via `toJSON()`):
  - `code: ErrorCode`
  - `category: ErrorCategory`
  - `message: string`
  - `statusCode: number`
  - `retryable: boolean`
  - `details?: Record<string, unknown>`

## Error Codes
- Canonical enum values:
  - `UNKNOWN`, `INTERNAL_ERROR`, `VALIDATION_FAILED`, `UNAUTHORIZED`, `FORBIDDEN`, `NOT_FOUND`, `CONFLICT`, `RATE_LIMITED`
  - `TOKEN_EXPIRED`, `REFRESH_FAILED`
  - `PROVIDER_UNAVAILABLE`, `QUOTA_EXCEEDED`
  - `CHUNK_FAILED`, `TRANSFER_FAILED`
  - `SHARE_ABUSE_LIMIT`, `VAULT_LOCKED`
- Default HTTP mappings are provided by `getDefaultHttpStatus()`.

## Edge Cases
- Unknown codes/messages fall back to `UNKNOWN` defaults.
- `AppError.fromUnknown()` wraps native `Error` and non-Error values into an `AppError`.
- Category fallback defaults to `system` when code mapping is missing.

## Example Payload
```json
{
  "code": "TOKEN_EXPIRED",
  "category": "auth",
  "message": "Session expired",
  "statusCode": 401,
  "retryable": false,
  "details": {
    "provider": "google"
  }
}
```
