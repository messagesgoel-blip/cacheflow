# Contract: 4.4

## File Paths

No HTTP endpoints are introduced by this task. The contract covers two new server-side modules:

- `lib/providers/vps/sshConnectionManager.ts` — `SSHConnectionManager` class + `sshConnectionManager` singleton
- `lib/providers/vps/VPSAdapter.ts` — `VPSAdapter` class implementing `ProviderAdapter` + `vpsAdapter` singleton

`VPSAdapter` satisfies all five `PROVIDER_PARITY_CHECKLIST` checks (`auth_lifecycle`, `file_discovery`, `file_mutation`, `stream_transfer`, `resumable_transfer`).

## sshConnectionManager.ts

### Public API

```ts
class SSHConnectionManager extends EventEmitter {
  constructor(options?: SSHConnectionManagerOptions)

  acquire(config: SSHAuthConfig): Promise<ManagedSSHSession>
  release(sessionId: string): void
  destroy(sessionId: string): void
  stats(): Array<{ id, alive, refCount, connectedAt, lastUsedAt }>
  shutdown(): Promise<void>
}

interface SSHAuthConfig {
  id: string           // unique key for LRU (e.g. `${userId}:${host}:${port}:${username}`)
  host: string
  port: number
  username: string
  password?: string    // mutually exclusive with privateKey
  privateKey?: string  // PEM-encoded; mutually exclusive with password
  passphrase?: string  // optional passphrase for encrypted private key
  connectTimeoutMs?: number  // default 10 000
}

interface ManagedSSHSession {
  id: string
  alive: boolean       // reflects underlying socket liveness
  client: ssh2.Client  // MUST NOT be .end()-ed by caller
  connectedAt: string  // ISO-8601
  refCount: number     // callers must release() after use
}

interface SSHConnectionManagerOptions {
  maxSessions?: number  // default 20
  idleTtlMs?: number    // default 300 000 (5 min)
}
```

### LRU Policy

- Maximum `maxSessions` live connections. On capacity breach, the least-recently-used session whose `refCount === 0` is evicted (connection closed).
- Background sweep runs every 60 s. Sessions idle for `idleTtlMs` with `refCount === 0` are evicted.
- Concurrent `acquire()` calls for the same `id` collapse to a single in-flight `connect()` Promise — no duplicate TCP handshakes.

### Events

| Event | Payload | Description |
|-------|---------|-------------|
| `evict` | `{ id: string }` | LRU eviction (capacity exceeded) |
| `idle-evict` | `{ id: string, idleMs: number }` | Idle TTL sweep eviction |
| `close` | `{ id: string }` | Underlying SSH socket closed |

### Credential Encoding

VPS credentials are encoded as a base64 JSON blob stored in `ProviderAuthState.accessToken`. This avoids modifying the existing auth pipeline. The blob shape is:

```ts
interface VPSCredentials {
  host: string
  port: number
  username: string
  authType: 'password' | 'key'
  password?: string
  privateKey?: string
  passphrase?: string
  rootPath: string
}
```

`accountId` is derived as `${userId}:${host}:${port}:${username}` and used as the LRU session key.

### Auth Lifecycle

```ts
connect(request: ConnectRequest): Promise<ConnectResponse>
// request.code must be a base64-encoded VPSCredentials JSON string.
// Opens an SSH connection to verify credentials; if it succeeds the
// connection is kept alive in the LRU pool.
// ConnectResponse.auth.accessToken = base64-encoded VPSCredentials
// ConnectResponse.auth.expiresAt = undefined (VPS SSH sessions do not expire)

refreshAuth(request: RefreshAuthRequest): Promise<RefreshAuthResponse>
// Always throws REFRESH_FAILED — VPS credentials do not expire.

validateAuth(request: ValidateAuthRequest): Promise<ValidateAuthResponse>
// Acquires an SSH session and stats rootPath via SFTP.
// Returns { valid: true } on success.
// Returns { valid: false, reason: 'revoked' | 'unknown' } on failure.

disconnect(request: DisconnectRequest): Promise<void>
// Calls sshConnectionManager.destroy(auth.accountId) to close and evict the session.
```

### File Discovery

```ts
listFiles(request: ListFilesRequest): Promise<ListFilesResponse>
// folderId defaults to VPSCredentials.rootPath.
// Returns all entries in the directory via SFTP readdir.
// hasMore is always false (no pagination).

searchFiles(request: SearchFilesRequest): Promise<SearchFilesResponse>
// Runs: find '<dir>' -maxdepth 5 -name '*<query>*' -not -path '*/.*' | head -100
// Returns ProviderFile with size=0 (no stat per result — performance tradeoff).
// hasMore is always false.

getFile(request: GetFileRequest): Promise<GetFileResponse>
// fileId is a full absolute remote path (e.g. /home/user/docs/file.txt).
// Uses SFTP stat to retrieve size and mtime.
```

### File Mutation

```ts
createFolder(request: CreateFolderRequest): Promise<CreateFolderResponse>
// Uses SFTP mkdir. parentId defaults to rootPath.
// Returns the new folder as ProviderFile with isFolder=true.

moveFile(request: MoveFileRequest): Promise<MoveFileResponse>
// Uses SFTP rename. newName overrides the source filename if provided.

copyFile(request: CopyFileRequest): Promise<CopyFileResponse>
// Runs: cp -r '<src>' '<dst>' via SSH exec.
// supportsServerSideCopy=false in descriptor because cp is a shell fallback,
// not a native SFTP server-side copy.

renameFile(request: RenameFileRequest): Promise<RenameFileResponse>
// Delegates to moveFile with the same parent directory.

deleteFile(request: DeleteFileRequest): Promise<void>
// Uses SFTP stat to detect directory vs file, then rmdir or unlink accordingly.
```

### Stream Transfer

```ts
downloadStream(request: DownloadStreamRequest): Promise<DownloadStreamResponse>
// Returns an SFTP createReadStream for the remote file.
// Supports byte-range via request.range: { start, end? }.
// The SFTP subsystem and SSH session are released when the stream closes or errors.
// contentLength = file size (or end - start + 1 for ranged requests).

uploadStream(request: UploadStreamRequest): Promise<UploadStreamResponse>
// Pipes request.stream into an SFTP createWriteStream at the target path.
// parentId defaults to rootPath. fileName is required.
```

### Resumable Upload

```ts
createResumableUpload(request: CreateResumableUploadRequest): Promise<CreateResumableUploadResponse>
// Stores session metadata in a process-local Map (sufficient for task scope).
// providerUploadId = full remote path.
// expiresAt = now + 24 hours.
// nextOffset starts at 0.

uploadResumableChunk(request: UploadResumableChunkRequest): Promise<UploadResumableChunkResponse>
// Appends payload at request.offset via SFTP createWriteStream with { flags: 'r+', start }.
// Returns { committedOffset, completed, session.nextOffset }.
// completed=true when isFinalChunk===true OR committedOffset >= contentLength.
// On completion the in-process session entry is deleted.

getResumableUploadStatus(request): Promise<GetResumableUploadStatusResponse>
// Returns current nextOffset from the in-process session map.

finalizeResumableUpload(request): Promise<FinalizeResumableUploadResponse>
// Stats the remote file to confirm it was written. Clears the session entry.

abortResumableUpload(request): Promise<void>
// Removes the in-process session entry. Does NOT delete the partial remote file
// (the caller is responsible for cleanup via deleteFile if needed).
```

### Share Links

Both `createShareLink` and `revokeShareLink` always throw `VALIDATION_FAILED` with the message "VPS/SFTP does not support share links."

## Error Codes

All errors are thrown as `AppError` with canonical `ErrorCode` values:

| Condition | ErrorCode |
|-----------|-----------|
| SSH auth failure (`authentication` in error message) | `UNAUTHORIZED` |
| Connection timeout / network failure | `PROVIDER_UNAVAILABLE` |
| Malformed base64 credential token | `UNAUTHORIZED` |
| SFTP: file not found (code 2) | `NOT_FOUND` |
| SFTP: permission denied (code 3) | `FORBIDDEN` |
| SFTP: file already exists (code 4) | `CONFLICT` |
| SFTP: other error | `TRANSFER_FAILED` |
| SSH exec non-zero exit | `TRANSFER_FAILED` |
| Chunk write failure | `CHUNK_FAILED` |
| Resumable session not found | `NOT_FOUND` |
| Share link operations | `VALIDATION_FAILED` |
| refreshAuth called | `REFRESH_FAILED` |
| No password or privateKey provided | `UNAUTHORIZED` |
| ssh2 package missing at runtime | `PROVIDER_UNAVAILABLE` |

## Edge Cases

- **LRU deduplication**: If two concurrent `acquire()` calls arrive for the same `id` before the first connection completes, both await the same in-flight `Promise`. Only one TCP handshake occurs.
- **Dead sessions**: If a cached session's underlying socket is closed externally (e.g. server-side timeout), `alive` returns `false`. The next `acquire()` detects this, evicts the dead entry, and establishes a new connection.
- **refCount overflow prevention**: `release()` never decrements below 0. Idle sweep skips sessions with `refCount > 0`.
- **Credential security (SEC-1)**: `password`, `privateKey`, and `passphrase` are held only in the in-memory `SSHAuthConfig` object within the active LRU node. They are never written to disk, logged, or emitted via events. On eviction the LRU node is GC-eligible.
- **Path normalization**: A `rootPath` or `folderId` of `/` is handled correctly — `/${name}` not `//name`.
- **Shell injection in searchFiles / copyFile / quota**: User-controlled strings (`query`, `fileId`, `rootPath`) are single-quote–escaped via `str.replace(/'/g, "'\\''")` before interpolation into shell commands.
- **SFTP subsystem per-operation**: Each operation opens a new SFTP subsystem and calls `sftp.end()` on completion. This avoids holding an SFTP channel open longer than necessary, reducing server-side resource usage.
- **`refreshAuth` not supported**: `supportsAuthRefresh: false` in the descriptor. Callers should never call `refreshAuth` on this adapter; it always throws `REFRESH_FAILED`.
- **Resumable sessions are process-local**: The `resumableSessions` Map is in-process only. If the API process restarts mid-upload, the session entry is lost. Callers should call `getResumableUploadStatus` to discover `nextOffset` before resuming; if a `NOT_FOUND` error is returned they must start a new session.
- **`abortResumableUpload` does not delete partial file**: Only the in-process session entry is removed. The partial remote file remains on the VPS. If cleanup is required, the caller must issue a separate `deleteFile` call.
- **`shutdown()` on process exit**: Call `sshConnectionManager.shutdown()` in process shutdown handlers to cleanly close all SSH connections and stop the idle-sweep timer.
- **ssh2 peer requirement**: `ssh2` must be added to the package's `dependencies`. The adapter uses `require('ssh2')` at runtime (lazy load) so the server process will throw `PROVIDER_UNAVAILABLE` with a clear message if the package is absent.

## Environment Variables Required

None. SSH credentials are passed directly through `request.code` (at connect time) and then stored in `ProviderAuthState.accessToken`. The `ssh2` npm package must be installed as a runtime dependency.

## Dependency Required

```
npm install ssh2
npm install --save-dev @types/ssh2
```
