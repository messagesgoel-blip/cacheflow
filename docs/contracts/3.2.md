# Contract: 3.2 — SSE Transfer Progress

## Endpoint URL

```
GET /api/transfers/[id]/progress
```

`[id]` is the BullMQ job ID (e.g. `transfer-user123-1700000000000`).

## Authentication

Cookie-based. The endpoint reads the `accessToken` httpOnly cookie and verifies it as a JWT against `JWT_SECRET`. No Bearer header is used.

## Request Shape

| Parameter | Source | Type | Required | Description |
|-----------|--------|------|----------|-------------|
| `id` | URL path | `string` | Yes | Transfer job ID |
| `accessToken` | Cookie | `string` (JWT) | Yes | Auth token |

No request body.

## Response — SSE Stream

**Headers:**
```
Content-Type: text/event-stream
Cache-Control: no-cache, no-transform
Connection: keep-alive
X-Accel-Buffering: no
```

**Event types** (all use `data:` containing JSON):

### `connected`
Sent immediately on stream open.
```json
{
  "jobId": "transfer-user123-1700000000000",
  "userId": "user123"
}
```

### `progress`
Sent for each `ProgressUpdate` emitted by the bridge (status `progress`, `completed`, or `failed`).
```json
{
  "jobId": "transfer-user123-1700000000000",
  "jobType": "transfer",
  "userId": "user123",
  "progress": 47,
  "status": "progress",
  "timestamp": 1700000003000,
  "data": {
    "userId": "user123",
    "sourceProvider": "gdrive",
    "destProvider": "onedrive",
    "fileId": "file456",
    "fileName": "report.pdf",
    "fileSize": 2097152,
    "operation": "copy"
  }
}
```

### `done`
Sent once the job reaches a terminal state (`completed` or `failed`). The stream closes immediately after.
```json
{
  "status": "completed"
}
```
or
```json
{
  "status": "failed"
}
```

### Keepalive (SSE comment)
Sent every 15 s to prevent proxy/load-balancer timeout:
```
: keepalive
```

## Error Codes

| HTTP status | Condition |
|-------------|-----------|
| `400` | Missing or empty `[id]` path segment |
| `401` | Missing `accessToken` cookie, or JWT verification failed (expired / invalid) |

No `500` — internal errors are logged server-side; the stream silently closes rather than exposing internal state to the client.

## Edge Cases

1. **Client disconnect**: The `request.signal` `abort` event fires; the subscription is cleaned up and the keepalive timer is cleared — no memory leak.
2. **Transfer already completed before connection**: No `progress` events will arrive. Clients should poll the transfers API for terminal state if the SSE stream closes without a `done` event.
3. **Redis unavailable**: `progressEmitter.initialize()` will throw; the route returns `401`-equivalent silent failure — callers should treat an immediate stream close as a transient error and retry.
4. **Concurrent connections**: `ProgressEmitter` delegates to `ProgressBridge.subscribeToJob` which maintains a `Set` per key; multiple SSE connections for the same job are supported.
5. **Unknown `jobId`**: The stream opens and stays open (receiving `connected`) but no `progress` events arrive. The keepalive runs until the client disconnects.
6. **JWT `id` field required**: The JWT payload must contain an `id` field (string). Malformed payloads return `401`.

## Example SSE Payload (wire format)

```
event: connected
data: {"jobId":"transfer-user123-1700000000000","userId":"user123"}

: keepalive

event: progress
data: {"jobId":"transfer-user123-1700000000000","jobType":"transfer","userId":"user123","progress":25,"status":"progress","timestamp":1700000001000}

event: progress
data: {"jobId":"transfer-user123-1700000000000","jobType":"transfer","userId":"user123","progress":75,"status":"progress","timestamp":1700000003000}

event: progress
data: {"jobId":"transfer-user123-1700000000000","jobType":"transfer","userId":"user123","progress":100,"status":"completed","timestamp":1700000005000,"result":{"success":true,"jobId":"transfer-user123-1700000000000","fileName":"report.pdf","fileSize":2097152,"duration":5000}}

event: done
data: {"status":"completed"}

```

## Implementation Files

| File | Role |
|------|------|
| `app/api/transfers/[id]/progress/route.ts` | Next.js App Router GET handler — auth, SSE stream, client lifecycle |
| `lib/transfers/progressEmitter.ts` | Thin wrapper around `progressBridge` — delegates subscriptions, exposes `onJobProgress` / `onUserProgress` |
| `lib/transfers/progressBridge.ts` | Redis pub/sub bridge (task 0.7) — upstream dependency |
