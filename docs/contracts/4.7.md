# Contract 4.7: Share Link Creation with 2FA Enforcement

## Endpoint
`POST /api/share` - Create a new share link

## Request Shape
```json
{
  "fileId": "string (required) - ID of the file to share",
  "password": "string (optional) - Password to protect the link",
  "expiresAt": "string (optional) - ISO date string for link expiry",
  "maxDownloads": "number (optional) - Max allowed downloads"
}
```

## Response Shape
Success (201):
```json
{
  "token": "string - Unique share token",
  "passwordRequired": "boolean - Whether password is required",
  "expiresAt": "string | undefined - Expiry date if set",
  "maxDownloads": "number | undefined - Max downloads if set"
}
```

## Error Codes
- `401`: Unauthorized - User not logged in
- `403`: Forbidden - 2FA not enabled (2FA-1 requirement)
- `400`: Bad Request - Invalid input parameters
- `404`: Not Found - File doesn't exist or unauthorized access
- `500`: Internal Server Error - Unexpected error

## Validation Rules
- User must have 2FA enabled to create share links (2FA-1)
- fileId must exist and belong to the requesting user
- maxDownloads must be a positive number if provided
- expiresAt must be a future date if provided

## Edge Cases
- If user doesn't have 2FA enabled, reject with 403 error
- If file doesn't belong to user, reject with 404 error
- If password is provided, it gets hashed before storage
- Token generation uses cryptographically secure random values

## Example Payload
Request:
```json
{
  "fileId": "abc123-uuid-here",
  "password": "securePassword123",
  "expiresAt": "2023-12-31T23:59:59.000Z",
  "maxDownloads": 10
}
```

Response:
```json
{
  "token": "a1b2c3d4e5f6...",
  "passwordRequired": true,
  "expiresAt": "2023-12-31T23:59:59.000Z",
  "maxDownloads": 10
}
```

## Additional Endpoints
`GET /api/share?token={token}` - Retrieve share link details (requires authentication)
`DELETE /api/share?token={token}` - Delete a share link (requires authentication)

## 2FA Scope Enforcement
- All share links require the creator to have 2FA enabled
- This applies regardless of whether the link has a password or not
- Enforces security policy outlined in SHARE-1
