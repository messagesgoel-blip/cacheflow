# VPS/SFTP Key Lifecycle (Task 6.6)

## New Endpoints

### Generate SSH Key Pair
- **Endpoint:** `POST /api/providers/vps/generate-key`
- **Description:** Generates a new Ed25519 SSH key pair for use with a VPS provider.
- **Request Body:** None
- **Response Shape:**
  ```json
  {
    "publicKey": "ssh-ed25519 AAAAC3... cacheflow-vps",
    "privateKey": "-----BEGIN OPENSSH PRIVATE KEY-----\n..."
  }
  ```
- **Error Codes:**
  - `500`: Failed to generate SSH key pair.

### Get Public Key
- **Endpoint:** `GET /api/providers/vps/:id/public-key`
- **Description:** Retrieves the OpenSSH formatted public key for an existing VPS connection.
- **Response Shape:**
  ```json
  {
    "publicKey": "ssh-ed25519 AAAAC3... cacheflow-vps"
  }
  ```
- **Error Codes:**
  - `404`: VPS connection not found.
  - `500`: Failed to retrieve public key.

## Updated Endpoints

### Create VPS Connection
- **Endpoint:** `POST /api/providers/vps`
- **Description:** Now supports providing `pemText` in the request body as an alternative to `pemFile` multipart upload.
- **Request Body (Partial):**
  ```json
  {
    "label": "My Node",
    "host": "1.2.3.4",
    "username": "root",
    "pemText": "-----BEGIN OPENSSH PRIVATE KEY-----\n..."
  }
  ```

### Update VPS Connection
- **Endpoint:** `PATCH /api/providers/vps/:id`
- **Description:** Now supports updating the credential via `pemText` in the request body.

## Library Changes

### SSHConnectionManager
- **File:** `lib/providers/vps/sshConnectionManager.ts`
- **Changes:** Added `hostFingerprint` verification to `SSHAuthConfig`. Connections will now fail if the server's fingerprint does not match the saved fingerprint (if provided).

### CredentialVault
- **File:** `lib/vault/credentialVault.ts`
- **Changes:** Added `generateSSHKey()` and `getPublicKeyFromPrivate()` methods.
