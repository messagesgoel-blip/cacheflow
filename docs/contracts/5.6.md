# Contract 5.6: Vault Access Gate - TOTP/PIN Unlock

## Endpoint
`POST /api/vault/[id]/unlock`

## Request Shape
```json
{
  "totp_code": "string (optional)",
  "pin": "string (optional)"
}
```

## Response Shape
```json
{
  "success": boolean,
  "session_token": "string (optional)",
  "expires_at": "timestamp (optional)"
}
```

## Error Codes
- 400: Invalid request (missing both totp_code and pin, or invalid format)
- 401: Invalid TOTP code or PIN
- 403: Vault access denied
- 404: Vault not found
- 429: Rate limited (too many attempts)

## Edge Cases
- Both totp_code and pin provided - validate either one
- Neither totp_code nor pin provided - return 400
- Invalid vault ID format - return 404
- Expired vault session - return 401
- Too many failed attempts - return 429

## Example Payload
Request:
```json
{
  "totp_code": "123456"
}
```

Response:
```json
{
  "success": true,
  "session_token": "vault_session_xyz",
  "expires_at": "2023-12-01T10:00:00Z"
}
```
