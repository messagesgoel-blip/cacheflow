# Contract: 1.5
## Endpoint or File Path
- `web/lib/vault/tokenVault.ts`
- `prisma/migrations/002_multi_account/migration.sql`

## Request / Response Shape
- Multi-account behavior in vault functions:
  - `addAccount()` appends account metadata (`accountLabel`, `accountOrder`, `isDefault`).
  - `getProviderAccounts()` returns sorted accounts for a provider.
- Migration adds/uses `oauth_tokens` fields:
  - `account_label`, `account_order`, `is_default`, `migrated_at`
  - indexes for `(user_id, provider, account_order)` and default-account uniqueness

## Error Codes
- Library-level validation errors:
  - `Maximum 3 accounts allowed per provider`
  - `Maximum 15 total accounts allowed`
- DB uniqueness/index constraints reject duplicates and conflicting defaults.

## Edge Cases
- Provider account ordering is re-normalized after `removeAccount()`.
- First account for provider is default when no default exists.
- Migration removes legacy unique constraint before creating new partial unique index.

## Example Payload
```json
{
  "provider": "dropbox",
  "accountLabel": "Work",
  "credentials": {
    "accessToken": "..."
  },
  "isDefault": false
}
```
