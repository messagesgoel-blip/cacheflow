# Contract 4.12: Remote Upload v1

## Endpoint
POST /api/remote-upload

## Request Shape
```json
{
  "url": "string", // HTTP/HTTPS URL to download from
  "provider": "string", // Target provider (e.g., "aws_s3", "gcp", "azure")
  "filename": "string", // Optional filename override
  "metadata": {
    "key": "value" // Optional metadata
  }
}
```

## Response Shape
```json
{
  "success": boolean,
  "fileId": "string", // ID of uploaded file
  "provider": "string", // Provider used
  "size": number, // File size in bytes
  "contentType": "string", // MIME type
  "uploadedAt": "string" // ISO timestamp
}
```

## Error Codes
- 400: Invalid URL, unsupported provider, invalid request format
- 403: Unauthorized access
- 429: Rate limited
- 500: Internal server error, provider unavailable
- 502: Failed to download from remote URL
- 504: Timeout downloading from remote URL

## Edge Cases
- Large files (streaming required)
- Invalid/malformed URLs
- Redirects in source URLs
- Authentication required for source URL
- Provider quota limitations
- Network timeouts
- Unsupported file types

## Example Payload
```json
{
  "url": "https://example.com/image.jpg",
  "provider": "aws_s3",
  "filename": "my-image.jpg",
  "metadata": {
    "userId": "12345",
    "category": "profile"
  }
}
```
