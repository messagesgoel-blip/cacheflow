# Contract: 0.1
## Endpoint or File Path
- `lib/providers/ProviderAdapter.interface.ts`
- `lib/providers/types.ts`

## Request / Response Shape
- Adapter identity:
  - `descriptor: ProviderDescriptor`
  - `ProviderDescriptor = { id, displayName, authType, capabilities }`
- Every adapter method accepts typed request objects with:
  - `context: { requestId, userId, abortSignal? }`
  - `auth: { accountId, accessToken, refreshToken?, expiresAt?, scopes? }` for authenticated operations
- Auth lifecycle:
  - `connect(ConnectRequest) -> ConnectResponse`
  - `refreshAuth(RefreshAuthRequest) -> RefreshAuthResponse`
  - `validateAuth(ValidateAuthRequest) -> ValidateAuthResponse`
  - `disconnect(DisconnectRequest) -> Promise<void>`
- File operations:
  - `listFiles`, `searchFiles`, `getFile`, `createFolder`, `moveFile`, `copyFile`, `renameFile`, `deleteFile`
- Transfer and zero-disk:
  - `downloadStream(DownloadStreamRequest) -> { file, stream: Readable, contentLength? }`
  - `uploadStream(UploadStreamRequest) -> { file }`
  - Resumable contract:
    - `createResumableUpload(...) -> { session }`
    - `uploadResumableChunk(...) -> { session, committedOffset, completed }`
    - `getResumableUploadStatus(...) -> { session }`
    - `finalizeResumableUpload(...) -> { session, file }`
    - `abortResumableUpload(...) -> Promise<void>`
- Share contract:
  - `createShareLink(...) -> { link }`
  - `revokeShareLink(...) -> Promise<void>`

## Error Codes
The adapter error shape is `ProviderAdapterError` with:
- `code`, `message`, `retryable`, optional `statusCode`, `providerRequestId`, `details`
- Allowed `code` values:
  - `UNAUTHORIZED`, `TOKEN_EXPIRED`, `REFRESH_FAILED`
  - `RATE_LIMITED`, `PROVIDER_UNAVAILABLE`, `QUOTA_EXCEEDED`
  - `CHUNK_FAILED`, `TRANSFER_FAILED`
  - `NOT_FOUND`, `CONFLICT`, `VALIDATION_FAILED`

## Edge Cases
- Expired auth mid-transfer: adapter returns `TOKEN_EXPIRED` or `REFRESH_FAILED`; caller can retry with refreshed auth.
- Out-of-order or duplicate chunk upload: adapter must preserve `session.nextOffset` and return authoritative `committedOffset`.
- Provider without resumable backend support: still implements methods and returns deterministic `VALIDATION_FAILED`/`PROVIDER_UNAVAILABLE` with `retryable=false`.
- Zero-disk guarantee: `downloadStream`/`uploadStream` operate on `Readable` streams only; no temp file paths are part of the contract.
- Cancellation: callers pass `abortSignal` in `context`; adapter should fail fast and keep resumable state consistent.
