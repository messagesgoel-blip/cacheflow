# Contract: 6.2 — Transfer Logs Supervised Experience

## Overview

Persistent terminal view renders real-time BullMQ and worker events over SSE with zero polling.

## Endpoint

**Path:** `/api/jobs/progress`

**Method:** GET (SSE stream)

**Description:** Server-Sent Events stream for real-time job progress updates.

## Request Shape

### Query Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `jobId` | string | Yes | Job ID to monitor |

## Response Shape

### Success (200 OK - SSE)

```typescript
interface JobProgressEvent {
  type: 'progress' | 'complete' | 'error' | 'heartbeat';
  jobId: string;
  payload?: {
    progress: number;
    total?: number;
    message?: string;
    timestamp: string;
  };
  error?: string;
}
```

## Edge Cases

1. ** jobs complete:**
   - Stream emits `complete` event with final stats
2. **Job fails:** Stream emits `error` event with message
3. **Connection drops:** Client reconnects automatically
4. **Expired credentials:** Stream emits `error` with 401

## Example Payload

```
event: progress
data: {"type":"progress","jobId":"job_123","payload":{"progress":50,"total":100,"message":"Transferring..."}}

event: complete
data: {"type":"complete","jobId":"job_123","payload":{"total":100,"duration":"5m32s"}}

event: error
data: {"type":"error","jobId":"job_123","error":"Network timeout after 30s"}
```
