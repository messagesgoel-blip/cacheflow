# Contract: Redis Namespace Allocation

## File Path

`lib/redis/client.ts`

## Overview

This module provides a namespaced Redis client factory that enforces database separation across different subsystems (SSE, Transfer, Rate Limiting, etc.).

## API

### getRedisClient(namespace: RedisNamespace): Redis

Returns a Redis client instance connected to the designated database for the given namespace.

#### Parameters

| Name      | Type           | Description                          |
|-----------|----------------|--------------------------------------|
| namespace | `RedisNamespace`| The functional namespace (see below) |

#### RedisNamespace Values

| Value        | DB Index | Purpose                              |
|--------------|----------|--------------------------------------|
| `cache`      | 0        | General caching, session data        |
| `sse`        | 1        | Server-Sent Events stream state      |
| `transfer`   | 2        | Transfer queue, quota tracking       |
| `rate_limit` | 3        | Rate limiting counters               |
| `workers`    | 4        | Background job state                 |

#### Returns

`Redis` - An ioredis client instance connected to the appropriate database.

#### Errors

| Code                 | Condition                                    |
|----------------------|----------------------------------------------|
| INVALID_NAMESPACE    | Thrown if an unrecognized namespace is passed |

### getRedisNamespace(dbIndex: number): RedisNamespace | null

Resolves a database index to its namespace identifier.

#### Parameters

| Name     | Type   | Description            |
|----------|--------|------------------------|
| dbIndex  | number | Redis database index   |

#### Returns

`RedisNamespace` if a mapping exists, `null` otherwise.

## Usage Example

```typescript
import { getRedisClient, RedisNamespace } from 'lib/redis/client';

const sseClient = getRedisClient('sse');       // Uses DB 1
const transferClient = getRedisClient('transfer'); // Uses DB 2
```

## Edge Cases

- **Connection failure**: Client emits error event and logs; does not throw
- **Invalid namespace**: Throws `Error` with message `Invalid Redis namespace: <value>`
- **Reuse**: Same client instance is returned for repeated calls with same namespace
