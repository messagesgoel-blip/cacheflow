## Purpose

- Freeze the first executable-looking task map for Sprint 7 without opening implementation work early.
- Keep agent scopes, acceptance gates, and red-line boundaries explicit while live PR cleanup is still in progress.
- Give the existing Sprint 7 startup prompt pack a concrete decomposition source.

## Gate Model

| Gate | Definition | Red lines |
| --- | --- | --- |
| `ZKV-1` | Browser-generated key hierarchy, versioned ciphertext envelope, and no server-side persistence of raw vault keys or decrypted vault payloads at rest. | No plaintext vault file content or long-lived private key material may be stored server-side. |
| `RECOVERY-1` | Unlock, recovery-kit creation, acknowledgement, and irreversible-loss messaging are explicit, testable, and consistent across devices. | No silent lockout, hidden destructive state, or ambiguous recovery promises. |
| `MIGRATE-1` | Existing vault users can migrate or defer safely, with explicit rollback/failure behavior and no silent metadata leakage into Files, Search, or Share flows. | No forced migration, silent corruption, or locked-item leakage during mixed legacy/new states. |
