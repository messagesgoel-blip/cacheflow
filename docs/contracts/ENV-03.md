# ENV-03: Repo-Owned Provider/Account Baseline

## Goal
Define a stable bootstrap provider/account baseline for rerun. Document which provider/account/folders/files/state are required. Add a verification script or checklist that confirms the bootstrap account is in the expected state before rerun.

## Baseline State Requirements
For the `V1-4-RERUN` suite to execute deterministically, the repo-owned bootstrap account must meet the following criteria:
1. **Session**: The session must resolve successfully in `web/api/auth/session` with a valid `accessToken` minted from the bootstrap credentials.
2. **Providers**: At least one primary cloud provider (e.g., Google Drive or Dropbox) must be fully authenticated and connected for the seeded bootstrap account.
3. **Provider Health**: At least one provider must return `healthy` from `web/api/connections/health`.
4. **Data State**:
   - The root directory must contain the seeded fixture files (`normal-file.txt`, `test-document.pdf`, `report.docx`, `image.png`).
   - Quota must have at least 1GB of free space.
5. **Account Settings**: 2FA must be disabled initially, or `ENV-01` must be satisfied so the live rerun can complete its challenge.

## Verification Checklist
A helper script `scripts/verify-live-baseline.sh` is provided to check session validity, health, provider status, root fixtures, 2FA state, quota, and a live VPS test for connected VPS rows. If no access token is provided, the script uses `scripts/bootstrap-dev-auth.sh` to mint one from the repo-owned bootstrap credentials in `web/.env.live`.
Set `REQUIRE_VPS_CONNECTION=true` when the rerun should also enforce a healthy VPS connection.

## Blockers
- None external. The bootstrap credentials generated by `scripts/setup-live-env.sh` are sufficient for the baseline check.
