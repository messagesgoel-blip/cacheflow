# Contract: Task 1.16 — Connections Page + Sidebar from Single Server State

Gate: SYNC-1

## Endpoint

`GET /api/connections`

## Request

Headers:
- `Authorization: Bearer <token>` (required)
- Or session cookie (forwarded from Next.js middleware)

No query parameters.

## Response — Success (200)

```json
{
  "success": true,
  "data": [
    {
      "id": "remote-uuid",
      "provider": "gdrive",
      "accountKey": "user@example.com",
      "remoteId": "remote-uuid",
      "accountName": "Work Account",
      "accountEmail": "user@example.com",
      "accountLabel": "Work Account",
      "isDefault": false,
      "status": "connected",
      "lastSyncAt": "2026-03-04T00:00:00.000Z"
    }
  ]
}
```

`status` values: `connected` | `disconnected` | `error`

## Response — Error Codes

| HTTP | `error` value | Condition |
|------|---------------|-----------|
| 401  | `Authentication required` | No auth header or cookie present |
| 401  | `Unauthorized` | Backend returned 401 |
| 500  | `Failed to retrieve connections` | Unhandled exception |
| 5xx  | `Failed to fetch connections from server` | Backend non-2xx (propagates backend status) |

## Edge Cases

- Empty `data` array returned when user has no remotes (not an error).
- `disabled` remotes are mapped to `status: "disconnected"`.
- Expired remotes (`expires_at` < now) are mapped to `status: "error"`.
- Backend response shapes handled: `{ remotes: [] }`, `{ data: { remotes: [] } }`, `{ data: [] }`.
- API base URL resolution prefers non-loopback env vars in order: `API_INTERNAL_URL`, `CACHEFLOW_API_INTERNAL_URL`, `API_URL`, `http://cacheflow-api:8100`.

## Sync Contract (SYNC-1)

Both consumers — `app/connections/page.tsx` and `components/Sidebar.tsx` — derive their connection list from a single `GET /api/connections` fetch executed in `ConnectionsPage`. The page maps `ServerConnection[]` → `ConnectedProvider[]` and passes that array as a prop to `Sidebar`, ensuring both display identical state on every render.
