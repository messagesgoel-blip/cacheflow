# Contract: 6.3 — Scheduled Job Resumption

## Overview

Recurring user-scoped backup jobs run with browser closed and expose reliable execution history.

## Endpoint

**Path:** `/api/jobs/scheduled`

**Method:** GET, POST

**Description:** Scheduled job management API.

## Request Shape

### Query Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `jobType` | string | No | Filter by job type |

### Request Body (POST - create job)

```typescript
interface CreateJobRequest {
  name: string;
  jobType: 'backup' | 'sync' | 'cleanup';
  cronExpression: string;
  enabled?: boolean;
  throttle?: number;
}
```

## Response Shape

### Success (200 OK)

```typescript
interface ScheduledJob {
  id: string;
  name: string;
  jobType: string;
  cronExpression: string;
  enabled: boolean;
  lastRunAt?: string;
  nextRunAt?: string;
  createdAt: string;
  updatedAt: string;
  metadata?: Record<string, unknown>;
}
```

## Edge Cases

1. **Invalid cron:** Returns 400 with error message
2. **Missing name:** Returns 400
3. **404:** Job not found

## Example Payload

### Create a backup job

```
POST /api/jobs/scheduled
Content-Type: application/json

{
  "name": "Daily backup",
  "jobType": "backup",
  "cronExpression": "0 2 * * *",
  "enabled": true
}
```

### Response

```
HTTP/1.1 200 OK
Content-Type: application/json

{
  "id": "job_1234567890",
  "name": "Daily backup",
  "jobType": "backup",
  "cronExpression": "0 2 * * *",
  "enabled": true,

  "nextRunAt": "2026-03-10T02:00:00Z",
  "createdAt": "2026-03-09T12:00:00Z",
  "updatedAt": "2026-03-09T12:00:00Z",
  "metadata": {}
}
```
