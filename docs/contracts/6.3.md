# Contract: 6.3 — Scheduled Job Resumption + Bandwidth Throttle

## Overview

Recurring user-scoped backup jobs run with browser closed and expose reliable execution history. Bandwidth throttle limits transfer speeds.

## Endpoint

**Path:** `/api/jobs`

**Method:** GET, POST, PUT, DELETE

**Description:** Scheduled job management API with throttle support.

## Request Shape

### Request Body (POST - create job)

```typescript
interface CreateJobRequest {
  name: string;
  jobType: 'backup' | 'sync' | 'cleanup';
  cronExpression: string;
  enabled?: boolean;
  throttle?: {
    maxBytesPerSecond: number | null;
  };
}
```

## Response Shape

### Success (200 OK)

```typescript
interface ScheduledJob {
  id: string;
  name: string;
  jobType: string;
  cronExpression: string;
  enabled: boolean;
  throttle?: {
    maxBytesPerSecond: number | null;
  };
  lastRunAt?: string;
  nextRunAt?: string;
  createdAt: string;
  updatedAt: string;
  metadata?: Record<string, unknown>;
}
```

## Throttle Presets

| Preset | Bytes/s | Label |
|--------|---------|-------|
| Unlimited | `null` | No limit |
| Low | 1,048,576 | ~1 MiB/s |
| Moderate | 5,242,880 | ~5 MiB/s |
| Standard | 26,214,400 | ~25 MiB/s |
| High | 104,857,600 | ~100 MiB/s |

## Acceptance Criteria

| Criterion | Description |
|-----------|-------------|
| SCHED-2 | Recurring user-scoped backup jobs run with browser closed and expose reliable execution history |
| THROTTLE-1 | Bandwidth throttle limits transfer speeds |

## Implementation Notes

- Throttle is applied via a `Transform` stream in `streamTransfer.ts` that limits bytes per second by introducing delays between 64KB chunks
- Transfer jobs now include optional `throttle` field in job data
- Scheduled job worker injects throttle into transfer payload when executing
- **Scope**: Throttle is only applied to server-side transfers via BullMQ workers. This is the correct scope because:
  - All actual file transfers between providers happen server-side (streamTransfer)
  - Client-side operations (direct browser-to-provider uploads) don't go through the worker
  - The throttle UI is designed for scheduled/background jobs which run server-side
