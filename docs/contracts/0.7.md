# Contract: 0.7 - BullMQ → SSE Bridge via Redis Pub/Sub

## Overview

The ProgressBridge connects BullMQ job progress to SSE streams via Redis pub/sub, enabling real-time transfer progress updates to clients.

## File Path

- **Module**: `lib/transfers/progressBridge.ts`
- **Architecture**: `docs/architecture/sse-pattern.md`

## API

### `progressBridge`

Singleton instance of the ProgressBridge.

### `ProgressBridge.initialize(): Promise<void>`

Initialize Redis pub/sub connections. Must be called before publishing or subscribing.

### `ProgressBridge.publishProgress(update: ProgressUpdate): Promise<void>`

Publish a progress update to the appropriate Redis channel.

- **Parameters**: `update` - ProgressUpdate object
- **Returns**: Promise<void>

### `ProgressBridge.publishTransferProgress(jobId, userId, progress, data?): Promise<void>`

Convenience method for publishing transfer job progress.

- **Parameters**:
  - `jobId`: Job identifier
  - `userId`: User who owns the job
  - `progress`: Progress percentage (0-100)
  - `data`: Optional transfer job data

### `ProgressBridge.publishTransferComplete(jobId, userId, result): Promise<void>`

Publish transfer job completion.

### `ProgressBridge.publishTransferFailed(jobId, userId, error): Promise<void>`

Publish transfer job failure.

### `ProgressBridge.subscribeToJob(userId, jobId, onProgress, onError): () => void`

Subscribe to progress updates for a specific job.

- **Parameters**:
  - `userId`: User identifier
  - `jobId`: Job identifier
  - `onProgress`: Callback for progress updates
  - `onError`: Callback for errors
- **Returns**: Unsubscribe function

### `ProgressBridge.subscribeToUser(userId, onProgress, onError): () => void`

Subscribe to all progress updates for a user.

- **Returns**: Unsubscribe function

## Request/Response Shape

### ProgressUpdate

```typescript
interface ProgressUpdate {
  jobId: string;
  jobType: 'transfer' | 'rate_limit';
  userId: string;
  progress: number | Record<string, unknown>;
  status: 'progress' | 'completed' | 'failed';
  timestamp: number;
  data?: TransferJobData;
  result?: TransferJobResult;
  error?: string;
}
```

### TransferJobData

```typescript
interface TransferJobData {
  userId: string;
  sourceProvider: string;
  destProvider: string;
  fileId: string;
  fileName: string;
  fileSize: number;
  operation: string;
}
```

### TransferJobResult

```typescript
interface TransferJobResult {
  success: boolean;
  jobId: string;
  fileName: string;
  fileSize: number;
  duration: number;
  error?: string;
}
```

## Error Codes

| Error | Description |
|-------|-------------|
| `Redis connection error` | Failed to connect to Redis pub/sub |
| `Message parse error` | Invalid JSON in progress message |
| `Subscriber callback error` | Error in user-provided callback |

## Edge Cases

1. **Client disconnect**: Unsubscribe callback must be called to prevent memory leaks
2. **Redis connection lost**: ProgressBridge will log errors but not throw
3. **Missing userId**: Updates without userId cannot be routed to subscribers
4. **High-frequency updates**: Consider debouncing on the client side

## SSE Integration

Clients connect to an SSE endpoint that subscribes to the ProgressBridge:

```typescript
// Server-side SSE handler
app.get('/api/transfers/stream', async (req, res) => {
  const jobId = req.query.jobId as string;
  const userId = req.user.id;

  res.setHeader('Content-Type', 'text/event-stream');
  res.setHeader('Cache-Control', 'no-cache');
  res.setHeader('Connection', 'keep-alive');

  const unsubscribe = progressBridge.subscribeToJob(
    userId,
    jobId,
    (update) => {
      res.write(`data: ${JSON.stringify(update)}\n\n`);
    },
    (error) => {
      console.error('SSE error:', error);
    }
  );

  req.on('close', unsubscribe);
});
```
