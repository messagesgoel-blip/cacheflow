# Task 5.11: Global cross-provider search v1 — name/metadata, ephemeral cache

## Endpoint URL
- GET `/api/search`
- POST `/api/search`

## Request/Response Shape

### GET Request Parameters (query parameters)
- `query` (required): Search query string for name/metadata search
- `providerIds` (optional): Comma-separated list of provider IDs to search within
- `folderId` (optional): Folder ID to scope search within
- `cursor` (optional): Pagination cursor for next page
- `pageSize` (optional): Number of results per page (default: 50)

### POST Request Body
```json
{
  "query": "search term",
  "providerIds": ["google", "onedrive"],
  "folderId": "folder123",
  "cursor": "cursor-string",
  "pageSize": 50
}
```

### Response Shape
```json
{
  "success": true,
  "data": {
    "files": [
      {
        "id": "file-id",
        "name": "filename.ext",
        "isFolder": false,
        "size": 1024,
        "parentId": "parent-folder-id",
        "path": "/path/to/file",
        "mimeType": "application/pdf",
        "etag": "etag-value",
        "checksum": "checksum-value",
        "createdAt": "2023-01-01T00:00:00Z",
        "modifiedAt": "2023-01-01T00:00:00Z",
        "webUrl": "https://provider.com/file-url",

```

## Error Codes
- `401 Unauthorized`: No valid access token provided
- `400 Bad Request`: Missing required query parameter
- `500 Internal Server Error`: Search operation failed

## Edge Cases
- Empty search query returns error
- Non-existent provider IDs are ignored
- Providers with invalid credentials are skipped (added to providersFailed)
- Mixed success/failure across providers returns partial results
- Cache hit returns results without calling provider APIs
- Cache miss performs actual search and stores results
