## Upload Pipeline Contract

All upload entry points accept `originalFileName: string` and forward it unchanged to the provider API. No UUID, timestamp, or hash is injected into the persisted file name or path.

### `uploadStream(request: UploadRequest): Promise<UploadResult>`

```ts
interface UploadRequest {
  context: ProviderOperationContext
  auth: ProviderAuthState
  provider: ProviderAdapter
  parentId?: string
  originalFileName: string
  contentType?: string
  contentLength?: number
  stream: Readable
}

interface UploadResult {
  fileId: string
  fileName: string
  size: number
  mimeType?: string
}
```

### `createResumableUpload(request: ResumableUploadRequest): Promise<ResumableSessionResult>`

```ts
interface ResumableUploadRequest {
  context: ProviderOperationContext
  auth: ProviderAuthState
  provider: ProviderAdapter
  parentId?: string
  originalFileName: string
  contentType?: string
  contentLength: number
  chunkSize: number
  checksum?: string
}

interface ResumableSessionResult {
  session: ResumableUploadSession
}
```

### `uploadChunk(request: ChunkUploadRequest): Promise<ChunkResult>`

```ts
interface ChunkUploadRequest {
  context: ProviderOperationContext
  auth: ProviderAuthState
  provider: ProviderAdapter
  session: ResumableUploadSession
  offset: number
  chunk: Buffer
  isFinalChunk: boolean
}

interface ChunkResult {
  session: ResumableUploadSession
  committedOffset: number
  completed: boolean
}
```

### `getUploadStatus / finalizeUpload / abortUpload`

Thin wrappers around `ProviderAdapter` resumable methods — pass-through, no name mutation.

## Provider Adapters

### GoogleDriveAdapter

| Method | Provider API |
|---|---|
| `uploadStream` | `PUT https://www.googleapis.com/upload/drive/v3/files?uploadType=resumable` |
| `createResumableUpload` | Same — returns `location` header as `providerUploadId` |
| `uploadResumableChunk` | `PUT {uploadUrl}` with `Content-Range` |
| `getResumableUploadStatus` | `PUT {uploadUrl}` with `bytes */{total}` |
| `finalizeResumableUpload` | Auto-finalized on last 200/201 chunk response |

Required env vars: `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`, `GOOGLE_REDIRECT_URI`

### OneDriveAdapter

| Method | Provider API |
|---|---|
| `uploadStream` | `PUT https://graph.microsoft.com/v1.0/me/drive/root:/{fileName}:/content` |
| `createResumableUpload` | `POST .../createUploadSession` |
| `uploadResumableChunk` | `PUT {uploadUrl}` with `Content-Range` |
| `getResumableUploadStatus` | `GET {uploadUrl}` |
| `abortResumableUpload` | `DELETE {uploadUrl}` |

Required env vars: `ONEDRIVE_CLIENT_ID`, `ONEDRIVE_CLIENT_SECRET`, `ONEDRIVE_REDIRECT_URI`, `ONEDRIVE_TENANT` (optional, defaults to `common`)

### DropboxAdapter

| Method | Provider API |
|---|---|
| `uploadStream` | `POST https://content.dropboxapi.com/2/files/upload` |
| `createResumableUpload` | `POST .../upload_session/start` — returns `session_id` as `providerUploadId` |
| `uploadResumableChunk` | `POST .../upload_session/append_v2` / `.../upload_session/finish` (final chunk) |
| `getResumableUploadStatus` | Zero-byte append to check offset |

Required env vars: `DROPBOX_CLIENT_ID`, `DROPBOX_CLIENT_SECRET`, `DROPBOX_REDIRECT_URI`

## Error Codes

| Code | Meaning |
|---|---|
| `VALIDATION_FAILED` | `originalFileName` is empty or missing |
| `UNAUTHORIZED` | Access token rejected by provider |
| `TOKEN_EXPIRED` | Token expired; caller must refresh |
| `RATE_LIMITED` | Provider returned 429 |
| `PROVIDER_UNAVAILABLE` | Provider returned 5xx or unexpected status |
| `TRANSFER_FAILED` | Upload or move operation failed |
| `CHUNK_FAILED` | Individual chunk upload rejected |
| `NOT_FOUND` | File or folder ID does not exist |
| `CONFLICT` | File name collision (provider-side) |
| `INTERNAL_ERROR` | Missing env vars or unexpected missing response field |
| `REFRESH_FAILED` | Token refresh rejected |

All errors are thrown as `AppError` instances (see `lib/errors/AppError.ts`).

## Edge Cases

- **Empty `originalFileName`**: `assertOriginalName` throws `VALIDATION_FAILED` before any provider call.
- **Large files via `uploadStream`**: The stream is buffered in memory; callers with large payloads should prefer `createResumableUpload` + `uploadChunk`.
- **Google Drive finalization**: The drive auto-finalizes on the last chunk (200/201). `finalizeResumableUpload` returns a synthesized `ProviderFile` from session metadata when no separate finalize call is needed.
- **OneDrive copy**: `copyFile` issues `POST .../copy` then re-fetches the source item metadata (async copy with no body response).
- **Dropbox paths**: Dropbox uses path strings as IDs for move/copy/rename operations. `fileId` is expected to be a Dropbox path (e.g. `/Documents/report.pdf`).
- **Dropbox share link revocation**: `revokeShareLink` expects `linkId` to be the full shared-link URL (passed back from `createShareLink`).
- **Auth token missing**: All adapters read OAuth client credentials from env vars at call time; missing vars throw `INTERNAL_ERROR` immediately.

## Example Payload

### Stream upload

```ts
const result = await uploadStream({
  context: { requestId: 'req-abc', userId: 'user-1' },
  auth: { accountId: 'user@example.com', accessToken: 'ya29.xxx' },
  provider: new GoogleDriveAdapter(),
  parentId: '1BxiMVs0XRA5nFMdKvBdBZjgmUUqptlbs74OgVE2upms',
  originalFileName: 'quarterly-report.pdf',
  contentType: 'application/pdf',
  contentLength: 204800,
  stream: readableStream,
})

// result.fileName === 'quarterly-report.pdf'
// No UUID prefix in result.fileName
```

### Resumable upload

```ts
const { session } = await createResumableUpload({
  context: { requestId: 'req-xyz', userId: 'user-1' },

  provider: new OneDriveAdapter(),
  originalFileName: 'backup.tar.gz',
  contentType: 'application/gzip',
  contentLength: 10 * 1024 * 1024,
  chunkSize: 5 * 1024 * 1024,

const chunk1 = await uploadChunk({
  context: { requestId: 'req-xyz', userId: 'user-1' },
  auth: { accountId: 'user@example.com', accessToken: 'EwAoA8...' },
  provider: new OneDriveAdapter(),
  session,
  offset: 0,
  chunk: firstChunkBuffer,
  isFinalChunk: false,
})

const chunk2 = await uploadChunk({ ..., offset: 5242880, isFinalChunk: true })
// chunk2.completed === true
```
