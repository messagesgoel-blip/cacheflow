# Contract: 1.2
## Endpoint or File Path
- `web/app/api/auth/refresh/route.ts`
- Endpoint: `POST /api/auth/refresh`
- Health endpoint: `GET /api/auth/refresh`

## Request / Response Shape
- `POST /api/auth/refresh`
  - Request: no JSON body required; expects `refreshToken` cookie.
  - Success response:
    - `{ success: true, accessToken: string, expiresAt: number }`
    - Sets rotated `refreshToken` cookie (`HttpOnly`, `sameSite=lax`, `path=/api/auth`).
  - Failure response:
    - `{ success: false, error: string }`
- `GET /api/auth/refresh`
  - Response:
    - `{ success: true, endpoint: '/api/auth/refresh', status: 'active', configured: boolean }`

## Error Codes
- `401`: no refresh token or invalid refresh token.
- `500`: server config error (`JWT_SECRET` missing in production) or refresh failure.

## Edge Cases
- Development fallback secret is used when `JWT_SECRET` is missing outside production.
- Production mode requires `JWT_SECRET`; missing secret causes refresh failure path.
- Refresh token is rotated on each successful refresh.

## Example Payload
```json
{
  "success": true,
  "accessToken": "<jwt>",
  "expiresAt": 1760000000000
}
```
