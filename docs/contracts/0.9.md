# Contract: 0.9 - Stateful Deployment Guard

## Endpoint URL or File Path

- `scripts/check-deployment-target.sh`
- `docs/architecture/deployment-constraints.md`
- `.env.example`

## Request / Response Shape

This is a CI shell contract (not an HTTP endpoint).

### Request

- Invocation: `./scripts/check-deployment-target.sh`
- Inputs:
  - Environment variable: `DEPLOYMENT_TARGET`
  - Repository root files/manifests

### Response

- Success: exit code `0`, logs `PASS` message
- Failure: exit code `1`, logs one or more `ERROR` lines

## Error Codes

| Error Code | Condition |
| --- | --- |
| `DEPLOYMENT_TARGET_BLOCKED` | `DEPLOYMENT_TARGET` contains blocked serverless/edge target identifiers |
| `PROHIBITED_MANIFEST_VERCEL` | `vercel.json` exists in repository root |
| `PROHIBITED_MANIFEST_NETLIFY` | `netlify.toml` exists in repository root |
| `PROHIBITED_MANIFEST_EDGE` | `wrangler.*` or `serverless.*` exists in repository root |
| `LAMBDA_DEFINITION_IN_ROOT` | Lambda function definition detected in root deployment manifest |
| `DEPLOYMENT_POLICY_BLOCKED` | One or more policy violations were found; gate must fail |

## Edge Cases

1. `DEPLOYMENT_TARGET` is unset: script defaults to `docker` and still validates repository manifests.
2. Case-insensitive target values: checks normalize to lowercase before policy evaluation.
3. Multiple violations at once: script reports all violations before failing.
4. Lambda manifest exists but has no Lambda resource definitions: not flagged as Lambda definition violation.
5. `wrangler.*` or `serverless.*` in repo root is always treated as prohibited edge/serverless config.

## Example Payload

```json
{
  "command": "./scripts/check-deployment-target.sh",
  "env": {
    "DEPLOYMENT_TARGET": "docker"
  },
  "response": {
    "exitCode": 0,
    "message": "PASS: DEPLOYMENT_TARGET='docker' and no prohibited serverless/edge manifests were found."
  }
}
```

```json
{
  "command": "./scripts/check-deployment-target.sh",
  "env": {
    "DEPLOYMENT_TARGET": "vercel"
  },
  "response": {
    "exitCode": 1,
    "errors": [
      {
        "code": "DEPLOYMENT_TARGET_BLOCKED",
        "message": "DEPLOYMENT_TARGET 'vercel' is not allowed. Use a long-running runtime (e.g. docker, kubernetes, vm)."
      },
      {
        "code": "DEPLOYMENT_POLICY_BLOCKED",
        "message": "Blocked by deployment policy for SSE-1 and TRANSFER-1."
      }
    ]
  }
}
```
