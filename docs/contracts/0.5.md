## Request / Response Shape

### BackpressureConfig
```typescript
interface BackpressureConfig {
  highWatermark: number;  // Buffer size to trigger backpressure
  lowWatermark: number;   // Buffer size to release backpressure
}

interface PipelineOptions {
  backpressure?: BackpressureConfig;
  onFlush?: () => void | Promise<void>;
}
```

### StreamPipeline Class
```typescript
class StreamPipeline<T> {
  write(data: T): Promise<void>;      // Write data to buffer
  flush(): Promise<void>;             // Process buffered data
  end(): Promise<void>;               // Signal end and flush
  abort(err: Error): Promise<void>;   // Abort with error
  getMetrics(): PipelineMetrics;      // Get current metrics
  readonly paused: boolean;           // Whether backpressure is applied
}

interface PipelineMetrics {
  buffered: number;   // Current buffer size
  processed: number;  // Total items processed
  dropped: number;    // Items dropped due to backpressure
  paused: boolean;    // Currently paused
}
```

### pipeline() Factory
```typescript
function pipeline<T>(
  source: AsyncIterable<T>,
  options: StreamPipelineFactoryOptions<T>
): StreamPipeline<T>;

interface StreamPipelineFactoryOptions<T> extends PipelineOptions {
  onData: (data: T) => void | Promise<void>;
  onError?: (error: Error) => void | Promise<void>;
  onEnd?: () => void | Promise<void>;
}
```

### createBackpressureConfig()
```typescript
function createBackpressureConfig(
  highWatermark: number,
  ratio?: number  // default 0.1
): BackpressureConfig;
```

## Behavior

- **Lazy Processing**: Data is buffered on `write()` but not processed until `flush()` is called or `end()` signals completion.
- **Backpressure**: When buffer reaches `highWatermark`, `paused` becomes `true` and new writes are dropped until buffer is drained via `flush()`.
- **Auto-release**: When `flush()` drains buffer to `lowWatermark`, `paused` becomes `false` and `onFlush` callback is invoked.
- **Metrics**: Always available via `getMetrics()`.

## Error Codes
- Uses `ErrorCode.VALIDATION_FAILED` when `highWatermark` is not positive.
- Custom errors propagated through `onError` handler.

## Edge Cases
- Zero or negative `highWatermark` throws validation error.
- `lowWatermark` defaults to at least 1 (minimum buffer floor).
- When paused, incoming writes are dropped (counted in `metrics.dropped`).
- Calling `end()` twice is idempotent.
- Pipeline handles async iterables that throw during iteration.
- Calling `flush()` when not paused processes all buffered items.

## Example Payload
```typescript
// Creating a pipeline with backpressure
const config = createBackpressureConfig(1000, 0.1);
// config = { highWatermark: 1000, lowWatermark: 100 }

const source = async function*() {
  for (let i = 0; i < 10000; i++) {
    yield { id: i, data: `item-${i}` };
  }
};

const p = pipeline(source(), {
  backpressure: config,
  onData: async (item) => {
    await processItem(item);
  },
  onError: (err) => {
    console.error('Pipeline error:', err.message);
  },
  onEnd: () => {
    console.log('Pipeline complete');
  },
  onFlush: () => {
    console.log('Backpressure released');
  },
});

// Manual flush for backpressure control
while (p.paused) {
  await p.flush();
  await new Promise(r => setTimeout(r, 100));
}

// Or just wait for end() to flush automatically
```
