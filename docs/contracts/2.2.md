# Contract 2.2: File Upload with Progress and Toast

## Endpoint URL
`/api/remotes/[uuid]/upload`

## Request/Response Shapes

### Upload File (POST)
**Request:**
- Method: `POST`
- Path: `/api/remotes/{provider_uuid}/upload`
- Headers: `Authorization: Bearer {token}`
- Body: `multipart/form-data` with:
  - `file`: File object to upload
  - `parentId`: (Optional) Parent folder ID
  - `fileName`: (Optional) Override filename

**Response:**
```json
{
  "success": true,
  "fileId": "string",
  "fileName": "string", 
  "size": number,
  "mimeType": "string",
  "jobId": "string",
  "sessionId": "string (for resumable uploads)"
}
```

### Get Upload Status (GET)
**Request:**
- Method: `GET`
- Path: `/api/remotes/{provider_uuid}/upload?jobId={jobId}&sessionId={sessionId}`
- Headers: `Authorization: Bearer {token}`

**Response:**
```json
{
  "success": true,
  "jobId": "string",
  "progress": number (0-100),
  "status": "waiting|active|completed|failed|delayed|uploading",
  "fileName": "string",
  "fileSize": number,
  "error": "string (if failed)"
}
```

### Upload Chunk (PUT)
**Request:**
- Method: `PUT`
- Path: `/api/remotes/{provider_uuid}/upload`
- Headers: `Authorization: Bearer {token}`
- Body: `multipart/form-data` with:
  - `chunk`: File chunk
  - `sessionId`: Resumable upload session ID
  - `offset`: Current offset
  - `isFinalChunk`: Boolean indicating if this is the final chunk

**Response:**
```json
{
  "success": true,
  "sessionId": "string",
  "committedOffset": number,
  "progress": number (0-100),
  "completed": boolean
}
```

## Error Codes
- `400`: Bad Request - Missing required fields
- `401`: Unauthorized - Invalid or missing token
- `404`: Not Found - Provider or upload session not found
- `500`: Internal Server Error - Upload failed
- `501`: Not Implemented - Feature not fully implemented

## Edge Cases
- Large files (>10MB) automatically use resumable uploads
- Small files use direct upload
- Failed uploads can be retried
- Progress tracking available for both direct and resumable uploads
- Auth token refresh happens automatically for 401 responses

## Implementation Notes
- Uses existing provider adapter system
- Integrates with job queue for progress tracking
- Supports both direct and resumable uploads based on file size
- Handles auth state retrieval for provider operations
