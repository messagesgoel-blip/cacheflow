# Contract: 4.2

## Endpoint or File Path

No HTTP endpoints are introduced by this task. The contract covers three new provider adapter modules:

- `lib/providers/box/index.ts` — `BoxAdapter` implementing `ProviderAdapter`
- `lib/providers/pcloud/index.ts` — `PCloudAdapter` implementing `ProviderAdapter`
- `lib/providers/yandex/index.ts` — `YandexAdapter` implementing `ProviderAdapter`

All three adapters conform to the `ProviderAdapter` interface defined in `lib/providers/ProviderAdapter.interface.ts` and satisfy all five `PROVIDER_PARITY_CHECKLIST` checks (`auth_lifecycle`, `file_discovery`, `file_mutation`, `stream_transfer`, `resumable_transfer`).

## Request / Response Shape

All adapters expose the same interface as every other provider adapter. Key type shapes per operation group:

### Auth Lifecycle

```ts
connect(request: ConnectRequest): Promise<ConnectResponse>
// ConnectRequest: { context, code?, callbackUrl?, codeVerifier?, state? }
// ConnectResponse: { account: ProviderAccountProfile, auth: ProviderAuthState }

refreshAuth(request: RefreshAuthRequest): Promise<RefreshAuthResponse>
// RefreshAuthRequest: { context, auth: ProviderAuthState }
// RefreshAuthResponse: { auth: ProviderAuthState, refreshedAt: string }
// Note: pCloud does not support token refresh (OAuth tokens are long-lived).
//       Calling refreshAuth on PCloudAdapter always throws REFRESH_FAILED.

validateAuth(request: ValidateAuthRequest): Promise<ValidateAuthResponse>
// ValidateAuthResponse: { valid: boolean, reason?: 'expired'|'revoked'|'insufficient_scope'|'unknown' }

disconnect(request: DisconnectRequest): Promise<void>
// Box: calls POST /oauth2/revoke with client credentials
// pCloud / Yandex: no-op (no server-side revocation required)
```

### File Discovery

```ts
listFiles(request: ListFilesRequest): Promise<ListFilesResponse>
// ListFilesRequest: { context, auth, folderId?, cursor?, pageSize? }
// ListFilesResponse: { files: ProviderFile[], nextCursor?: string, hasMore: boolean }

// Provider defaults for root folderId:
//   Box:    '0'       (Box numeric root folder ID)
//   pCloud: '0'       (pCloud numeric root folder ID)
//   Yandex: 'disk:/'  (Yandex Disk root path)

searchFiles(request: SearchFilesRequest): Promise<SearchFilesResponse>
// SearchFilesRequest: { context, auth, query, folderId?, cursor?, pageSize? }

getFile(request: GetFileRequest): Promise<GetFileResponse>
// GetFileRequest: { context, auth, fileId }
// GetFileResponse: { file: ProviderFile }
// Box: fileId is a numeric string (e.g. '12345678')
// pCloud: fileId is a numeric string (fileid or folderid)
// Yandex: fileId is a path string (e.g. 'disk:/Documents/file.pdf')
```

### File Mutation

```ts
createFolder(request: CreateFolderRequest): Promise<CreateFolderResponse>
moveFile(request: MoveFileRequest): Promise<MoveFileResponse>
copyFile(request: CopyFileRequest): Promise<CopyFileResponse>
renameFile(request: RenameFileRequest): Promise<RenameFileResponse>
deleteFile(request: DeleteFileRequest): Promise<void>
```

### Stream Transfer

```ts
downloadStream(request: DownloadStreamRequest): Promise<DownloadStreamResponse>
// DownloadStreamResponse: { file: ProviderFile, stream: Readable, contentLength?: number }
// All three providers resolve a download URL then stream the response body.
// Supports range requests via request.range: { start, end? }.

uploadStream(request: UploadStreamRequest): Promise<UploadStreamResponse>
// UploadStreamResponse: { file: ProviderFile }
// Box: multipart form upload to https://upload.box.com/api/2.0/files/content
// pCloud: binary POST to uploadfile endpoint
// Yandex: two-step (GET upload URL, then PUT body)
```

### Resumable Transfer

```ts
createResumableUpload(request): Promise<CreateResumableUploadResponse>
// Returns: { session: ResumableUploadSession }
// Box:    creates upload session at /files/upload_sessions
// pCloud: calls upload_create, stores uploadid in providerUploadId
// Yandex: calls /resources/upload for a resumable upload URL (providerUploadId = that URL)

uploadResumableChunk(request): Promise<UploadResumableChunkResponse>
// Returns: { session, committedOffset: number, completed: boolean }
// Box:    PUT /upload_sessions/{id} with Content-Range + Digest headers
// pCloud: POST upload_write with uploadoffset param
// Yandex: PUT to providerUploadId with Content-Range header

getResumableUploadStatus(request): Promise<GetResumableUploadStatusResponse>
// Returns: { session } with updated nextOffset

finalizeResumableUpload(request): Promise<FinalizeResumableUploadResponse>
// Box:    POST /upload_sessions/{id}/commit
// pCloud: GET upload_save with folderid + name params
// Yandex: GET /resources to fetch the committed file metadata

abortResumableUpload(request): Promise<void>
// Box:    DELETE /upload_sessions/{id}
// pCloud: GET upload_delete
// Yandex: no-op (upload URL expires naturally)
```

### Share Links

```ts
createShareLink(request: CreateShareLinkRequest): Promise<CreateShareLinkResponse>
// CreateShareLinkRequest: { context, auth, fileId, expiresAt?, password? }
// CreateShareLinkResponse: { link: ProviderShareLink }
// Box:    PUT /files/{id} with shared_link.access='open'
// pCloud: GET getfilepublink
// Yandex: PUT /resources/publish then GET /resources for public_url

revokeShareLink(request: RevokeShareLinkRequest): Promise<void>
// Box:    PUT /files/{id} with shared_link=null
// pCloud: GET deletepublink (requires linkId)
// Yandex: PUT /resources/unpublish (uses fileId, not linkId)
```

## External API Endpoints

### Box
- Auth: `https://api.box.com/oauth2/token`, `https://api.box.com/oauth2/revoke`
- REST API: `https://api.box.com/2.0`
- Upload API: `https://upload.box.com/api/2.0`

### pCloud
- Auth & API (US): `https://api.pcloud.com`
- Auth & API (EU): `https://eapi.pcloud.com` (selected automatically when `auth.accountId` starts with `'eu'`)
- Token exchange: `POST /oauth2_token`

### Yandex Disk
- OAuth: `https://oauth.yandex.com/token`
- User profile: `https://login.yandex.ru/info?format=json`
- Disk API: `https://cloud-api.yandex.net/v1/disk`

## Error Codes

All errors are thrown as `AppError` with canonical `ErrorCode` values:

| HTTP Status | ErrorCode             |
|-------------|-----------------------|
| 401         | UNAUTHORIZED          |
| 403         | FORBIDDEN             |
| 404         | NOT_FOUND             |
| 409         | CONFLICT              |
| 429         | RATE_LIMITED          |
| 507         | QUOTA_EXCEEDED        |
| 5xx         | PROVIDER_UNAVAILABLE  |
| chunk fail  | CHUNK_FAILED          |
| move/copy   | TRANSFER_FAILED       |
| no refresh  | REFRESH_FAILED        |
| bad input   | VALIDATION_FAILED     |

pCloud-specific numeric error codes are mapped internally:
- `1000` → `UNAUTHORIZED`
- `2009`, `2010` → `NOT_FOUND`
- `2004` → `CONFLICT`
- `4000` → `QUOTA_EXCEEDED`
- others → `PROVIDER_UNAVAILABLE` or `VALIDATION_FAILED`

## Edge Cases

### Box
- Root folder is `'0'`; callers must not pass an empty string as `folderId`.
- `getFile` first attempts `/files/{id}`, falls back to `/folders/{id}` on 404.
- Upload session commit (finalize) may return HTTP 202 (Accepted) which is treated as success.
- Share link revocation sets `shared_link: null` via PUT; no separate link ID is needed.
- `getResumableUploadStatus` calls the Box Upload API (`upload.box.com`), not the content API.

### pCloud
- `supportsAuthRefresh` is `false`; pCloud OAuth tokens do not expire.
- US vs EU API base is selected from `auth.accountId` prefix (`eu*` → EU endpoint).
- `downloadStream` first calls `getfilelink` to obtain a redirect-free CDN URL.
- `searchFiles` does not support pagination (`hasMore` is always `false`).
- `listFiles` paginates via offset over the full folder contents returned in a single call.
- `abortResumableUpload` calls `upload_delete`.

### Yandex Disk
- `fileId` values are full disk paths (e.g. `disk:/folder/file.txt`), not opaque IDs.
- Async operations (move, copy, delete) may return HTTP 202 with an operation `href`; the adapter polls until `status === 'success'` with a 1-second delay, up to 30 attempts.
- Resumable upload `providerUploadId` stores the upload URL (not a session ID string).
- `abortResumableUpload` is a no-op; Yandex upload URLs expire automatically.
- `revokeShareLink` uses `fileId` (path), not `linkId`.
- Avatar URL is constructed from `default_avatar_id` via `avatars.yandex.net`.
