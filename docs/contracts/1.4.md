# Contract: 1.4
## Endpoint or File Path
- `web/lib/vault/tokenVault.ts`
- `prisma/migrations/001_token_vault/migration.sql`

## Request / Response Shape
- Vault library shape:
  - `loadVault() -> { entries: VaultEntry[]; version: number; lastSyncAt?: number }`
  - `addAccount(account)`, `getAccount(provider, accountLabel?)`, `getProviderAccounts(provider)`
  - `removeAccount(provider, accountLabel)`, `setDefaultAccount(provider, accountLabel)`
  - `clearVault()`, `getVaultStats()`
- SQL schema (`tokens` table) includes:
  - identity/user/provider fields
  - `encrypted_credentials` + `encryption_version`
  - multi-account metadata (`account_label`, `account_order`, `is_default`)
  - indexes/constraints for active/default account lookup

## Error Codes
- Library-level thrown errors:
  - `Vault save failed`
  - `Account not found`
  - limit errors when account caps are exceeded
- SQL constraints enforce invalid state rejection (unique/default/check constraints).

## Edge Cases
- Vault load/decrypt failures return empty vault (`{ entries: [], version: 1 }`).
- Encryption key derives from `accessToken` cookie; falls back to `dev-key` when missing.
- Default account uniqueness is enforced by partial unique index.

## Example Payload
```json
{
  "provider": "google",
  "accountLabel": "Primary",
  "credentials": {
    "accessToken": "...",
    "refreshToken": "..."
  },
  "isDefault": true
}
```
