# Contract: 1.4

## File Paths
- `lib/vault/tokenVault.ts` — Token Vault library (created)
- `prisma/migrations/001_token_vault/migration.sql` — DB schema (existing)

## Public API

### Types
```ts
VaultCredentials { accessToken, refreshToken?, expiresAt?, scope?, ...extra }
VaultEntry       { provider, accountLabel, accountOrder, isDefault, credentials,
                   remoteId?, lastUsedAt?, createdAt, updatedAt, encryptionVersion }
VaultState       { entries: VaultEntry[], version: number, lastSyncAt?: number }
```

### Functions
```ts
encrypt(plaintext: string): string
  // → "<iv_hex>:<authTag_hex>:<ciphertext_hex>:<version>"

decrypt(ciphertext: string): string
  // Throws on tamper or wrong key (GCM auth failure)

getVault(): TokenVault    // process-level singleton
setVault(v: TokenVault)   // replace singleton (test isolation)
```

### TokenVault methods
| Method | Description |
|---|---|
| `loadVault()` | Returns `VaultState` snapshot |
| `hydrateVault(state)` | Replaces all entries from a saved state |
| `addAccount(entry)` | Add or overwrite account; enforces ≤3 per provider |
| `getAccount(provider, label?)` | Lookup by label; label="default" → default account |
| `getProviderAccounts(provider)` | All accounts for provider, sorted by `accountOrder` |
| `removeAccount(provider, label)` | Delete entry; throws if not found |
| `setDefaultAccount(provider, label)` | Mark as default; clears previous default |
| `clearVault()` | Reset all state |
| `getVaultStats()` | `{ totalAccounts, providers, defaultAccounts }` |
| `serialise()` | `encrypt(JSON(loadVault()))` → store in `tokens.encrypted_credentials` |
| `deserialise(ciphertext)` | Decrypt and hydrate; resets to empty vault on error |

## Error Conditions
| Error | Trigger |
|---|---|
| `TOKEN_ENCRYPTION_KEY is not set` | Key env var missing (SEC-1) |
| `Invalid ciphertext format` | `decrypt()` receives malformed string |
| GCM decryption error | Wrong key or tampered ciphertext |
| `Account not found` | `removeAccount` / `setDefaultAccount` on missing label |
| Max accounts exceeded | `addAccount` when provider already has 3 accounts |

## SQL Schema Summary (`tokens` table)
- `id UUID PK`, `user_id UUID FK → users`, `provider VARCHAR(50)`
- `encrypted_credentials BYTEA NOT NULL`, `encryption_version INT DEFAULT 1`
- `account_label VARCHAR(50)`, `account_order INT [1..3]`, `is_default BOOL`
- `is_active BOOL`, `expires_at TIMESTAMP`, `last_used_at TIMESTAMP`
- Unique constraint: `(user_id, provider, account_label)`
- Partial unique index: `(user_id, provider)` WHERE `is_default AND is_active`

## Edge Cases
- `deserialise()` catches all errors → empty vault (never throws to caller)
- `getAccount(provider, 'default')` resolves to the `isDefault=true` entry
- `addAccount` with existing label performs an upsert (preserves `createdAt`)
- Max 3 accounts per provider enforced in-memory; DB `CHECK(account_order IN [1..3])` backs it
