# Contract: 6.4 — Media Streaming Polish

## Overview

Implements HTTP range request support for seekable media playback and stable long-lived streaming without disk buffering or unbounded memory growth.

## Endpoints

### 1. Media Stream Proxy

**Path:** `/api/remotes/[uuid]/upload` (also handles at `/api/remotes/[uuid]/stream`)

**Methods:** GET, POST, HEAD

**Description:** Proxies media streaming from cloud providers with HTTP Range header support for seekable playback.

---

## Request Shape

### Query Parameters (GET)

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `fileId` | string | Yes* | File ID on the provider |
| `provider` | string | Yes* | Provider identifier (google, onedrive, dropbox, etc.) |
| `path` | string | No | File path (for VPS/WebDAV providers) |

*Either via query params or request body.

### Request Body (POST)

```typescript
interface StreamRequest {
  fileId: string;
  provider: string;
  path?: string;
}
```

### Headers

| Header | Required | Description |
|--------|----------|-------------|
| `Authorization` | Yes | Bearer token for authentication |
| `Range` | No | HTTP Range header for seekable playback (e.g., `bytes=0-`) |
| `Content-Type` | For POST | `application/json` |

---

## Response Shape

### Success (200 OK)

```typescript
interface StreamResponse {
  // Direct stream - no wrapper
  // Headers include:
  // - Content-Type: media MIME type
  // - Content-Length: size in bytes (for full file)
  // - Content-Range: bytes start-end/total (for partial content)
  // - Accept-Ranges: bytes
}
```

### Partial Content (206 Partial Content)

Returned when `Range` header is provided and server supports range requests.

**Headers:**
- `Content-Type`: media MIME type
- `Content-Length`: bytes in range
- `Content-Range`: `bytes start-end/total`
- `Accept-Ranges`: `bytes`

### Error Responses

```typescript
interface ErrorResponse {
  success: false;
  error: string;
}
```

| Status | Code | Description |
|--------|------|-------------|
| 400 | BAD_REQUEST | Missing fileId or provider |
| 401 | UNAUTHORIZED | Invalid or missing authentication |
| 404 | NOT_FOUND | File not found on provider |
| 502 | BAD_GATEWAY | Upstream provider unavailable |

## Stream Transfer Options

For backend transfers (`lib/transfers/streamTransfer.ts`):

```typescript
interface StreamTransferOptions {
  // ... existing fields
  /**
   * High water mark for the PassThrough stream.
   * Controls memory usage during streaming.
   * Default: 64 KiB (STREAM-1 compliance)
   */
  highWaterMark?: number;
}
```

**STREAM-1 Compliance:**
- Default `highWaterMark` of 64 KiB prevents unbounded memory growth
- Zero-disk transfer (pipe from source to target)
- Progress callbacks for monitoring
- Proper cleanup on abort

## Edge Cases

1. **Provider doesn't support Range requests:** Fall back to full download
2. **Large files (>10GB):** Streaming still works, but seek may require new range requests
3. **Network interruption:** Stream fails; client must restart from last position
4. **Unsupported provider:** Returns 400 BAD_REQUEST
5. **Expired credentials:** Returns 401 with option to refresh token
6. **Empty file:** Returns 200 with Content-Length: 0

## Example Payloads

### Request: Stream first 1MB of video

```
GET /api/remotes/[uuid]/stream?fileId=abc123&provider=google
Range: bytes=0-1048575
Authorization: Bearer <token>
```

### Response: Partial Content

```
HTTP/1.1 206 Partial Content
Content-Type: video/mp4
Content-Length: 1048576
Content-Range: bytes 0-1048575/52428800
Accept-Ranges: bytes

<binary data...>
```

### Request: Get full audio file

```
POST /api/remotes/[uuid]/stream
Content-Type: application/json
Authorization: Bearer <token>

{
  "fileId": "music.mp3",
  "provider": "onedrive"
}
```

### Response: Full Content

```
HTTP/1.1 200 OK
Content-Type: audio/mpeg
Content-Length: 5242880
Accept-Ranges: bytes

<binary data...>
```
