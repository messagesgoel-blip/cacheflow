# Contract 4.9: Abuse Controls — Rate Limits, Throttling, Link Access Logging

## Endpoint: Share Revocation with Abuse Controls

### URL
`POST /api/share/[id]/revoke`

### Request Shape
- Method: `POST`
- Path Parameter: `id` (string) - The unique identifier of the share to revoke
- Headers: 
  - `Content-Type`: `application/json`
  - `Authorization`: Bearer token (if required)
  - Client identification headers (IP, User-Agent, Referer)

### Response Shape
#### Success Response (200 OK)
```json
{
  "success": true,
  "message": "Share successfully revoked",
  "shareId": "<id of the revoked share>"
}
```

#### Error Responses
##### Rate Limit Exceeded (429 Too Many Requests)
```json
{
  "error": "Rate limit exceeded",
  "message": "Too many requests. Please try again in X seconds."
}
```

##### Share Not Found (404 Not Found)
```json
{
  "error": "Not Found",
  "message": "Share not found or you do not have permission to revoke it."
}
```

##### Internal Server Error (500 Internal Server Error)
```json
{
  "error": "Internal Server Error",
  "message": "Could not revoke share. Please try again." 
}
```

### Error Codes
- `429`: Rate limit exceeded - client has made too many requests within the time window
- `404`: Share not found or unauthorized access attempt
- `500`: Internal server error during revocation process

### Edge Cases
1. Invalid share ID format
2. Share already revoked
3. User not authorized to revoke the share
4. Concurrent revocation attempts
5. Database connectivity issues during revocation
6. Redis connectivity issues for rate limiting
7. Malformed request headers

### Example Payload
#### Request
No request body required, uses path parameter only.

#### Successful Response
```json
{
  "success": true,
  "message": "Share successfully revoked",
  "shareId": "abc123def456"
}
```

### Rate Limiting Configuration
- Default share access limit: 100 requests per 15 minutes per IP address
- Rate limit key: `share_revoke_{client_ip}`

### Access Logging
- Each access attempt is logged with:
  - Share ID
  - Client IP address
  - User agent string
  - Timestamp
  - Referrer (if available)
