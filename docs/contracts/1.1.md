# Contract: 1.1
## Endpoint or File Path
- `web/lib/apiClient.ts`
- `web/lib/interceptors/authInterceptor.ts`

## Request / Response Shape
- Client wrappers:
  - `apiClient.get<T>(endpoint, options?) -> Promise<{ success: boolean; data?: T; error?: string }>`
  - `apiClient.post<T>(endpoint, body?, options?) -> Promise<{ success: boolean; data?: T; error?: string }>`
  - `apiClient.delete<T>(endpoint, options?) -> Promise<{ success: boolean; data?: T; error?: string }>`
- Interceptor contract:
  - `authInterceptor(url, options?, maxRetries=1) -> Promise<Response>`
  - On 401, calls `POST /api/auth/refresh` with `credentials: include`, then retries request.

## Error Codes
- Uses HTTP status flow (not `ErrorCode` enum):
  - Non-OK responses return `{ success: false, error: "HTTP <status>: <statusText>" }`.
  - Refresh failure throws `Refresh failed with status <status>`.
  - Retry exhaustion throws `Session expired - redirecting to login` and redirects to `/login?reason=session_expired`.

## Edge Cases
- Concurrent 401s are serialized by a singleton `refreshPromise`.
- Network errors are retried until `maxRetries` is exceeded.
- `authFetch` ensures `credentials: include` unless explicitly overridden.

## Example Payload
```ts
const result = await apiClient.getConnections();
// { success: true, data: ProviderConnection[] }
// or { success: false, error: "HTTP 401: Unauthorized" }
```
