# Contract: 1.3
## Endpoint or File Path
- `web/lib/auth/cookieAuth.ts`
- `web/lib/tokenManager.ts`

## Request / Response Shape
- Cookie auth API:
  - `setAuthCookies(accessToken, refreshToken, user, options?)`
  - `getAccessToken()`, `getRefreshToken()`, `getUserData()`
  - `clearAuthCookies()`, `isAuthenticated()`, `getAuthHeaders()`
- Local storage migration API:
  - `migrateFromLocalStorage()` removes legacy `cf_` / `cacheflow_` keys.
- Token manager (browser state) supports provider-token CRUD with multi-account keys and settings.

## Error Codes
- No HTTP endpoint-level status codes in these files.
- Runtime errors thrown for token limits in `tokenManager`:
  - `MAX_PER_PROVIDER_REACHED:<provider>`
  - `MAX_TOTAL_REACHED`

## Edge Cases
- Refresh token cookie path is restricted to `/api/auth`.
- `getUserData()` safely returns `null` if cookie parsing fails.
- Migration helper removes legacy localStorage keys to reduce client-side token exposure.

## Example Payload
```ts
await setAuthCookies(accessToken, refreshToken, {
  userId: 'u_123',
  email: 'user@example.com',
  name: 'User'
});

const headers = await getAuthHeaders();
// { Authorization: 'Bearer <accessToken>' }
```
