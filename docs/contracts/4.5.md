## credentialVault.ts — Public API

### Credential payload types

```ts
interface VPSCredentialPayload {
  kind: 'vps';
  host: string;
  port: number;
  username: string;
  authType: 'password' | 'key';
  password?: string;      // only when authType === 'password'
  privateKey?: string;    // PEM-encoded; only when authType === 'key'
  passphrase?: string;    // optional passphrase for encrypted private key
  rootPath: string;
}

interface WebDAVCredentialPayload {
  kind: 'webdav';
  url: string;
  username: string;
  password: string;
  basePath?: string;
}

type CredentialPayload = VPSCredentialPayload | WebDAVCredentialPayload;

type EncryptedCredential = string;
// Wire format: "<iv_hex>:<authTag_hex>:<ciphertext_hex>:<version>"
// Four colon-delimited segments — all required for round-trip decryption.
```

### Functions

```ts
encryptCredential(payload: CredentialPayload): EncryptedCredential
decryptCredential(encrypted: EncryptedCredential): CredentialPayload
```

### Class

```ts
class CredentialVault {
  encrypt(payload: CredentialPayload): EncryptedCredential
  decrypt(encrypted: EncryptedCredential): CredentialPayload
  rotate(payload: CredentialPayload): EncryptedCredential
  summary(payload: CredentialPayload): Record<string, unknown>
  // summary() never includes password, privateKey, or passphrase
}

getCredentialVault(): CredentialVault  // module singleton
setCredentialVault(vault: CredentialVault): void  // test injection
```

## POST /api/connections/vps

### Authentication

Requires a valid JWT in either:
- `Cookie: accessToken=<jwt>`
- `Authorization: Bearer <jwt>`

Returns `401` if absent or invalid.

### Request body — VPS (SFTP)

```json
{
  "kind": "vps",
  "host": "192.168.1.1",
  "port": 22,
  "username": "admin",
  "authType": "password",
  "password": "s3cr3t",
  "rootPath": "/home/admin",
  "accountLabel": "Primary"
}
```

```json
{
  "kind": "vps",
  "host": "my-vps.example.com",
  "port": 22,
  "username": "deploy",
  "authType": "key",
  "privateKey": "-----BEGIN OPENSSH PRIVATE KEY-----\n...\n-----END OPENSSH PRIVATE KEY-----",
  "passphrase": "optional-passphrase",
  "rootPath": "/home/deploy",
  "accountLabel": "Production"
}
```

### Request body — WebDAV

```json
{
  "kind": "webdav",
  "url": "https://dav.example.com",
  "username": "user@example.com",
  "password": "app-password",
  "basePath": "/dav/",
  "accountLabel": "Nextcloud"
}
```

### Success response — 200

```json
{
  "success": true,
  "remoteId": "d1e2f3a4-...",
  "provider": "vps",
  "accountKey": "192.168.1.1:22:admin",
  "displayName": "admin@192.168.1.1:22"
}
```

The response contains **no credential fields** — only opaque identifiers (SEC-1).

### Field constraints

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `kind` | `'vps' \| 'webdav'` | Yes | Selects credential type |
| `host` | string | VPS only | Non-empty |
| `port` | number | VPS optional | 1–65535, default 22 |
| `username` | string | Yes | Non-empty |
| `authType` | `'password' \| 'key'` | VPS only | |
| `password` | string | VPS+password / WebDAV | Required for respective authType |
| `privateKey` | string | VPS+key only | PEM-encoded |
| `passphrase` | string | VPS+key optional | For encrypted private keys |
| `rootPath` | string | VPS optional | Default `/` |
| `url` | string | WebDAV only | Must be a valid URL |
| `basePath` | string | WebDAV optional | e.g. `/dav/` |
| `accountLabel` | string | Optional | Default `Primary` |

## Error codes

All error responses follow the shape:

```json
{
  "success": false,
  "error": "<human-readable message>",
  "code": "<SCREAMING_SNAKE>"
}
```

| HTTP status | `code` | Condition |
|-------------|--------|-----------|
| 400 | `VALIDATION_FAILED` | Missing/invalid fields or invalid JSON |
| 401 | `UNAUTHORIZED` | Missing/invalid JWT |
| 404 | `NOT_FOUND` | DELETE — remoteId not found or belongs to another user |
| 500 | `INTERNAL_ERROR` | Encryption failure (TOKEN_ENCRYPTION_KEY absent) or DB error |

---

## Edge cases

- **Upsert semantics** — POST uses `(userId, provider, accountKey)` as the unique key. Submitting the same host+port+username a second time overwrites the encrypted credential in place. `remoteId` is stable across updates.
- **accountKey derivation** — VPS: `"${host}:${port}:${username}"`. WebDAV: `"${url}:${username}"`. Two connections with different `accountLabel` but the same host/username share a single row (label is not part of the key — label is stored only in `displayName`).
- **TOKEN_ENCRYPTION_KEY absent** — `encryptCredential()` throws synchronously. The route catches this and returns `500 INTERNAL_ERROR`. Plaintext credentials are never written to DB or logs in this case.
- **GCM authentication failure** — `decryptCredential()` throws if the ciphertext has been tampered with or if the key has been rotated without re-encrypting stored values. Callers must handle this and treat affected rows as invalid.
- **Key rotation** — Use `CredentialVault.rotate(vault.decrypt(enc))` after updating `TOKEN_ENCRYPTION_KEY` to re-encrypt stored rows. The `keyVersion` column in `user_remotes` can be used to track which rows have been migrated.
- **Port range** — Ports outside 1–65535 are rejected with `VALIDATION_FAILED`. Port is defaulted to `22` if omitted.
- **Wrong provider for DELETE** — DELETE only removes rows where `provider IN ('vps', 'webdav')`. Attempting to delete an OAuth provider row via this endpoint returns `404`.
- **Concurrent upserts** — Prisma upsert is atomic at the DB level (`@@unique` constraint). Two concurrent POSTs for the same credential will serialize; the later one wins.

## Environment variables required

| Variable | Purpose |
|----------|---------|
| `TOKEN_ENCRYPTION_KEY` | Hex-encoded key (≥32 bytes). Used for AES-256-GCM. Shared with `tokenVault`. |
| `JWT_SECRET` | Used to verify the `accessToken` cookie / `Authorization` header. |
| `DATABASE_URL` | PostgreSQL connection string for Prisma. |
