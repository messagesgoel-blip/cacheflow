# Ideas Notepad

Purpose: capture candidate ideas, UI wishes, and future tasks without turning them into roadmap commitments.

Status:

- intake only
- non-canonical
- read-only for agent execution

Rules:

- only `docs/roadmap.md` is the canonical product roadmap
- ideas written here are not approved, scheduled, or committed roadmap work
- an item moves from this file into execution only after orchestrator agreement is reflected in `docs/roadmap.md`
- agents may read from this file and copy items into their own temporary notes or task decomposition
- agents must not edit this file directly or treat it as an executable task source

Promotion flow:

1. idea is captured here
2. orchestrators review and agree
3. roadmap is updated in `docs/roadmap.md`
4. only then may manifests, contracts, or sprint plans absorb the work
